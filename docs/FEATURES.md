# Features

---

## Voice Expense Logging

Users tap the microphone and speak their expense in natural language. Multiple expenses can be described in a single recording.

Example: "I spent RM18 on lunch and RM5 for parking."

The app records audio locally, sends it to the backend for transcription, then immediately runs AI extraction on the returned transcript.

---

## Type Expense Mode

Users who prefer not to use voice can switch to the Type tab and type their expense in plain English.

Example: `Bubble tea RM12, parking RM5`

Both voice and type modes use the same AI extraction pipeline.

---

## AI Transcription

Voice recordings are sent to the `transcribe-audio` Supabase Edge Function, which calls OpenAI Whisper to convert speech to text. The transcript is shown to the user before extraction begins so they can verify accuracy.

Model: configurable via Supabase secret `OPENAI_TRANSCRIBE_MODEL` (default: `gpt-4o-mini-transcribe`).

---

## AI Expense Extraction

The transcript or typed text is sent to the `extract-expense` Edge Function, which calls an OpenAI chat model with a structured JSON schema. The AI returns:

- Amount
- Merchant name
- Category (Food & Drinks, Transport, Utilities, Shopping, Entertainment, Health, Education, Other)
- Date (defaults to today if not specified)
- Confidence score (0–1, stored internally)
- Clarification question if the input is ambiguous

Multiple expenses can be extracted from a single input.

Model: configurable via Supabase secret `OPENAI_EXPENSE_MODEL` (default: `gpt-4o-mini`).

---

## Review and Edit Flow

Before any expense is saved, the user sees a Review screen with all extracted fields:

- Amount
- Merchant name
- Category (with a picker)
- Date
- Note (optional)

The user can edit any field. If multiple expenses were extracted, all are shown as a list and saved together via the **Save Expense** button.

---

## Authentication

- Email and password sign-up and login via Supabase Auth
- Remember Me — session is persisted via Expo SecureStore and restored on next launch
- Google Sign-In is prepared and can be enabled in a development or production build
- Sessions expire and are automatically refreshed by the Supabase client

---

## User Profile and Budget

- Editable display name (shown on Home as a greeting)
- Editable monthly budget (default: RM 2,000)
- Budget used and remaining are computed in real time from the current month's expenses

---

## Home Dashboard

- Monthly budget, spent, and remaining
- Budget percentage used
- Category donut chart with color-coded segments
- Category legend with percentage breakdown (matches donut colors)
- Recent expense list (last 3)
- Quick-add button that opens the Add Expense screen
- "Live totals" badge links to the Budget Alert page

---

## Expense History

- All saved expenses grouped by date (Today, Yesterday, or a specific date)
- Shows amount, merchant, category pill, and time for each entry
- Tap any expense to open the detail view

---

## Expense Detail

- Full view of all expense fields (amount, merchant, category, date, note, source, AI confidence)
- AI confidence shown as both a labeled row and a visual progress bar
- Delete expense
- Editing an existing expense is planned for a future release

---

## Budget Alert

- Visual progress bar showing budget usage
- Amount spent vs. total monthly budget
- Remaining balance highlighted
- Budget grade (A / B / C) based on usage percentage

---

## Insights Dashboard

### Spending Overview

Weekly bar chart comparing this month's spending week by week. Relative bar heights show which weeks had the most activity.

### Category Breakdown

- Donut chart with color-coded segments representing each spending category
- Legend with category name and percentage
- RM total in the center of the donut

### Budget Health

- Progress bar showing percentage of monthly budget used
- Remaining balance
- Budget grade badge

### AI Spending Insights

Unlocked after 3 or more expenses in the current month.

Generated by the `generate-insights` Edge Function using the user's real spending data. The AI analyzes aggregated stats (not raw expense rows) and returns:

**Summary** — headline and description with a tone (positive, neutral, or warning).

**Metric cards:**
- Budget Usage
- Projected Month-End Spend
- Spending Concentration
- Weekly Pattern

**Recommendations** — actionable suggestions with estimated impact.

**Spending Patterns** — behavioral observations about the user's spending.

Users can tap **Refresh insights** to regenerate at any time.
